Random Passwords

Generate cryptographically secure random passwords with full control over character sets, length, and ambiguity filtering.

Basic Usage

Python
from password_generator import generate

# Default: 16 characters, all character types
password = generate()
print(password)  # e.g. "k8#Qx!2mNp@Lw9Rz"

Custom Length

Python
# Short password (minimum 4)
short = generate(length=8)

# Long password (maximum 256)
long = generate(length=64)

Control Character Sets

Python
# Letters only (no digits or symbols)
letters_only = generate(digits=False, symbols=False)

# Alphanumeric (no symbols)
alnum = generate(symbols=False)

# Uppercase + digits only
strong = generate(lowercase=False, symbols=False)

# Symbols only (minimum 4 characters)
symbols_only = generate(uppercase=False, lowercase=False, digits=False, length=8)

Custom Symbols

Python
# Use only specific symbols
password = generate(symbol_chars="!@#$%")

# Avoid potentially dangerous symbols in shell
safe = generate(symbol_chars="-_.~")

Exclude Ambiguous Characters

Characters like l, I, 1, O, 0 look similar in some fonts. Exclude them:

Python
# No ambiguous characters
password = generate(length=20, exclude_ambiguous=True)
# Never contains: l, I, 1, O, 0, o

Using GeneratorConfig

For reusable configurations:

Python
from password_generator import generate, GeneratorConfig

# Create a config for your app's requirements
app_config = GeneratorConfig(
    length=24,
    uppercase=True,
    lowercase=True,
    digits=True,
    symbols=True,
    exclude_ambiguous=True,
)

# Generate multiple passwords with the same config
for i in range(5):
    print(generate(app_config))

Parameters

How It Works

  1. Pool seeding — One character is drawn from each enabled pool to guarantee diversity
  2. Filling — Remaining characters drawn from a combined pool
  3. Fisher-Yates shuffle — Cryptographically secure shuffle for uniform distribution
Security: Uses Python's secrets module, which is cryptographically secure. Never use the random module for passwords.
